# Dual-Fast-v28 设计方案

## 一、问题背景

Dual-Fast 是一个求解 **最小权支配集问题 (Minimum Weight Dominating Set, MWDS)** 的求解器。原版采用 "对偶下界 + 局部搜索" 的框架：通过迭代计算对偶下界（LB）来引导搜索方向，同时用局部搜索不断优化上界（UB），最终缩小 LB-UB 之间的 gap。

**原版的不足：**
1. 下界构造过程中节点选择完全依赖贪心启发式，缺乏历史学习能力
2. 局部搜索采用固定的 BMS 采样量，对不同规模的实例缺乏适应性
3. 没有对实例难度进行分类，简单和困难实例使用相同策略
4. 当下界收敛后仍然持续计算下界，浪费时间预算
5. 搜索陷入局部最优时缺乏有效的逃脱机制

## 二、核心设计思路

v28 的改进围绕一个中心思想：**让下界计算和上界搜索形成闭环反馈，并根据实例特征自适应地分配计算资源。**

具体通过以下四个模块实现：

### 模块 1：AQ 蚁群信息素强化学习

**动机：** 原版每轮下界构造是独立的，不会利用之前轮次中"哪些节点经常出现在好的解中"这一信息。如果某个节点频繁出现在高质量下界解中，下一轮应该优先考虑它。

**设计：**
- 为每个节点维护一个信息素值 AQ[v]，初始化为 1.0
- 每轮下界构造时，用 K 只"蚂蚁"并行构造 K 个不同的下界解
- 每只蚂蚁以概率 q0 选择启发式最优节点（利用），以概率 1-q0 按启发式值做轮盘赌选择（探索）
- 选择最优蚂蚁的解作为本轮结果
- 对全局信息素做蒸发（衰减），对最优蚂蚁解中的节点做正反馈（增强）
- 信息素值被嵌入启发式公式 HEUR(node) 中，形成 `信息素^指数 * 原启发值` 的乘积

**参数：**
- K = 5（蚂蚁数量，可调）
- rho = 0.05（蒸发率，即每轮信息素衰减 5%）
- q0 = 0.9（贪心选择概率，90% 贪心 + 10% 随机探索）
- beta = 3.5（信息素权重指数，控制信息素对选择的影响力）
- tau_min = 0.01, tau_max = 20.0（信息素上下界，防止过度集中或消散）

### 模块 2：上下界协同反馈

**动机：** 下界和上界本质上在解决同一个问题的两面。好的上界解中出现的节点，往往也是下界构造中应该重视的节点。

**设计：**
- 当局部搜索找到新的全局最优上界时，将解中的节点反馈给信息素系统
- 反馈强度与上界改善幅度成正比：`bonus = (初始UB - 当前最优UB) / 初始UB * 0.5`
- 上界构造（compute_upper_bound）中也加入信息素加权，但权重较低（beta * 0.3），避免过度偏向

这样形成了一个正向循环：好的下界引导好的上界 → 好的上界反馈给信息素 → 信息素引导更好的下界。

### 模块 3：三层自适应策略

**动机：** 不同实例的难度差异巨大。简单实例（gap < 1%）花大量时间在蚁群优化上是浪费；困难实例（gap > 50%）需要更重度的学习。

**设计：** 在第一轮下界计算后，根据 gap 和图密度将实例分为三档：

| 层级 | 条件 | 策略 | RL时间预算 |
|------|------|------|-----------|
| Tier-1 Easy | gap < 1% 且 density ≤ 1.0 | 不启用 AQ，纯贪心 | 0% |
| Tier-2 Medium | gap ≤ 50% 且 density ≤ 10.0 | 标准 AQ，K=5 | 15% |
| Tier-3 Hard | 其余 | 重度 AQ，K=8~10，rho=0.08 | 25% |

RL 阶段结束后，将蚂蚁数量降为 1，但保留信息素继续引导后续的单蚂蚁构造。

### 模块 4：LB 收敛检测与多重启

**动机：** 下界计算存在天花板——当 LB 连续多轮不再改善时，继续计算只是浪费时间。应该把剩余时间全部用于上界搜索。

**设计：**
- 追踪 LB 停滞计数器：每轮如果 LB 没有改善就 +1，改善了就归零
- 当停滞 ≥ 3 次且已用时间 > 40% 时，判定 LB 收敛
- 将剩余时间切换为纯上界搜索模式：
  - 先做一次正常搜索（40% 剩余时间）
  - 然后循环多重启：每次用"扰动初始化"产生新的初始解，再跑局部搜索
  - 扰动初始化以 15% 概率随机跳过已知最优解中的部分节点，然后贪心修复

## 三、辅助改进

### 自适应 BMS 采样
- 原版：固定采样 50~60 个候选节点
- v28：采样量 = max(50, 解规模的 10%) + 随机扰动
- 对大实例自动增大采样范围，提高找到好移除节点的概率

### 多移除扰动
- 当搜索超过 200 步未改善，且当前有冗余节点（score=0）时
- 额外随机移除 1~3 个冗余节点，加大扰动力度
- 仅在 USE_ADVANCED 模式（|V| ≥ 800）下启用

### 停滞自适应蒸发率
- LB 改善时：降低蒸发率（rho *= 0.9），让信息素更持久
- LB 停滞时：增大蒸发率（rho *= 1.1），加速探索新方向
- 停滞达到 10 次时：部分重置信息素（保留 30% 旧值 + 70% 初始值）

### Bug 修复
- 修复了 vertex 指针数组分配时 sizeof(int) 应为 sizeof(int*) 的内存 bug
- 修复了 OPTIMAL 标志在跨轮次间未重置的 bug
- 修复了 FIRST_GAP 被反复覆盖的 bug

## 四、整体流程对比

### 原版流程
```
读图 → 预处理 → 循环 {计算下界 → 构造初始解 → 局部搜索} → 输出
```

### v28 流程
```
读图(自动格式检测) → 预处理 → 难度分类(三层)
→ [RL阶段] 多蚂蚁下界计算(信息素更新)
→ 循环 {
    检查LB收敛?
    → 已收敛: 多重启上界搜索(扰动初始化) → 结束
    → 未收敛: 计算下界(信息素引导) → 构造初始解 → 局部搜索(AQ平局打破+多移除)
              → 发现更优解? → 反馈信息素
  }
→ 输出
```
